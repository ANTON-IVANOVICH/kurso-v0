### Интернационализация новых экранов (🌐 `kurso-web`)

### Айдентика: фавиконки, PWA и обложки (🌐 `kurso-web`, ⚙️ `kurso-admin`, 🤝 `kurso-partner`)

### Партнёрская программа и виджеты (🔧 `kurso-api`, 🌐 `kurso-web`)

- Партнёрский (реферальный) кабинет: у каждого пользователя своя ссылка
  (`/?ref=code`) с UTM-метками. Реальная атрибуция: при переходе по ссылке
  ставится cookie `kurso_ref` (90 дней), которую читает клик-аут (`ref_code` в
  `clickouts`) и регистрация (кого пригласил пользователь). Статистика живая —
  клики, регистрации, разбивка по UTM-меткам и график по дням; заработок —
  честная оценка по переходам до появления учёта подтверждённых обменов.
- Конструктор виджетов: пять видов (таблица курсов, калькулятор, бейдж,
  карточка, кнопка) с настройками (тема, акцент, резерв, логотип, UTM),
  **живым превью на реальных курсах** и генерацией кода для встраивания
  (HTML / React / Markdown / WordPress) с реф-меткой пользователя.
- Backend: у пользователей появились `referral_code` и `referred_by`
  (миграция `00013_user_referrals`); клик-аут пишет `ref_code`; защищённый
  `GET /api/v1/auth/partner` отдаёт код и реальную статистику. В kurso-web —
  плагин `ref.client` (захват метки в cookie) и авторизованный fetch с
  прозрачным refresh.

### Карта обменников на Mapbox (🔧 `kurso-api`, 🌐 `kurso-web`)

- Иммерсивная полноэкранная карта (вариант 2) с плавающими стеклянными
  панелями: поиск, фильтр направления, переключатель «Карта / Список», зум и
  «моё местоположение». На мобиле сохраняется нижняя навигация.
- Реальный **Mapbox GL**: тёмная карта с маркерами-ценниками обменников,
  поповером выбранной точки и маршрутом. Токен задаётся через
  `NUXT_PUBLIC_MAPBOX_TOKEN`; без него страница показывает статичный превью с
  точками, спроецированными по реальным координатам (все панели и карусель
  работают). Геолокация браузера даёт «моё местоположение» и расстояния
  (Haversine); «Перейти» — реальный клик-аут через `/go/{slug}`.
- Backend: у обменников появились координаты и адрес
  (миграция `00012_exchanger_geo`), заполняемые для сид-обменников (реальные
  адреса в Москве). Публичный `GET /api/v1/map?direction=` отдаёт обменники с
  координатами и их текущим курсом по направлению.

### Умный алерт и история курса (🔧 `kurso-api`, 🌐 `kurso-web`)

- Конструктор алерта переведён на вариант «умного алерта»: правило читается
  одной строкой («Уведомить, когда USDT → Тинькофф станет выше 81.50 ₽»), с
  живым графиком курса за 7 дней и прогнозом срабатывания, посчитанным по
  **реальной** истории (на сколько нужно изменение и сколько раз за неделю курс
  доходил до порога). Каналы, фильтры и создание алерта — как прежде.
- Backend: публичный эндпоинт истории курса
  `GET /api/v1/rates/{direction}/history?days=` — лучший курс по бакетам из
  `rates_history` (даунсэмплинг до ~60 точек). Питает график и прогноз.

### Кабинет обменника (🔧 `kurso-api`, 🤝 `kurso-partner`)

Backend:

- Авторизация представителей обменника (`exchanger_users`, bcrypt): эндпоинты
  `/partner/auth/login|refresh|logout|me`, middleware `RequireMerchant`,
  который резолвит обменник представителя. Токены как у админки: access — в
  памяти клиента, refresh — в **httpOnly-cookie** (`kurso_partner_rt`).
  Auth-сервис обобщён на третью личность (админ / пользователь / обменник),
  каждая со своим секретом; сид-представитель заводится при старте.
- API кабинета (всё на живых данных обменника):
  - `GET /partner/dashboard` — метрики (активные/устаревшие курсы, клики
    сегодня и вчера, рейтинг, неотвеченные отзывы) и трафик за 7 дней;
  - `GET /partner/rates` + `POST /partner/rates/{direction}/refresh` — курсы по
    направлениям со статусом фида; ручное обновление перештампывает фид и
    републикует направление в общий поток (публичный сайт видит свежесть сразу);
  - `GET /partner/reviews` + `POST /partner/reviews/{id}/reply` — отзывы
    обменника и один ответ на отзыв (`review_replies`, проверка принадлежности);
  - `GET /partner/traffic?days=` — клики по направлениям и дневной ряд из
    `clickouts` (7/14/30 дней);
  - `GET /partner/profile` + `PATCH` — описание, сайт и название; изменение
    отправляет карточку на повторную модерацию;
  - `GET /partner/complaints` — жалобы на отзывы обменника;
  - `GET /partner/billing` — модель CPA: оценка за текущий месяц по реальным
    кликам + история выплат (`partner_payouts`).

kurso-partner:

- SPA на **PrimeVue** (та же тёмная тема, что и админка), плотный десктопный
  кабинет с сайдбаром и живыми бейджами (устаревшие фиды, неотвеченные отзывы,
  открытые жалобы). Vue Router с защищёнными роутами, Pinia, fetch-клиент с
  прозрачным `401 → refresh → повтор`.
- Страницы: вход с 2FA-полем, главная (метрики, отзывы к ответу, здоровье фида,
  трафик), курсы (обновление фида по направлению и всех сразу), отзывы (фильтры
  и редактор ответа со счётчиком), трафик (график, разбивка по направлениям,
  экспорт CSV), профиль (редактирование с пометкой о повторной модерации),
  жалобы, биллинг (накопление за месяц, тариф, история платежей).

### Каталог курсов (🔧 `kurso-api`, 🌐 `kurso-web`)

Backend:

- Публичный API каталога: `GET /api/v1/directions`, `/exchangers`,
  `/exchangers/{slug}`, `/rates/{direction}`. Карточка обменника отдаёт агрегаты
  по активным курсам: суммарный резерв, число направлений, торгуемые активы, год
  добавления.
- Живой поток курсов через SSE: `GET /api/v1/rates/{direction}/stream`
  (in-process pub/sub hub, дедупликация подписок на слаг).
- Клики и редиректы: `GET /go/{slug}?direction=` — запись в `clickouts` и редирект
  на реферальную ссылку обменника.
- Сидинг демонстрационного каталога (валюты, направления, обменники, курсы) и
  фоновый тикер обновления курсов.

kurso-web:

- Каркас приложения: layout, шапка, футер, мобильная нижняя навигация.
- Главная: живой калькулятор обмена (строгий числовой ввод с группировкой,
  свап сторон, счётчик обменников и свежесть данных, реакция на SSE) и рабочие
  фильтры/сортировка.
- Каталог обменников (адаптивная таблица/список) и страница обменника (рейтинг,
  метрики, направления с клик-аутом, блок отзывов).
- Подключение к API через Pinia Colada + SSE (ref-counted stream — один
  `EventSource` на слаг).
- Дровер выбора валюты, поиск по сайту, панель уведомлений.
- Юридические и контентные страницы с боковой навигацией, системные страницы
  (404/500/maintenance), заглушка карты, страницы входа и онбординга.
- RU/EN интернационализация с переключателем языка (каталоги сообщений,
  плюрализация, cookie-персистентность, реактивный `<html lang>`).
- Дизайн-токены, базовые UI-компоненты, хелперы форматирования чисел и валют.

### Админка каталога (⚙️ `kurso-admin`, 🔧 `kurso-api`)

kurso-admin:

- SPA на **PrimeVue** (тёмная тема через `definePreset`: primary — фирменный
  синий, поверхности — тёмная палитра макета). Vue Router с защищёнными роутами,
  Pinia (`auth` + `catalog`), fetch-клиент с bearer-токеном.
- Страница входа (email/пароль + код 2FA), подключённая к реальной авторизации
  (см. раздел ниже).
- Каркас с сайдбаром (группы «Обзор», «Каталог», «Модерация»,
  «Монетизация · Данные»), мобильный drawer, блок пользователя с выходом.
- Дашборд: метрики, блок «Требуют внимания», здоровье парсеров (`DataTable`),
  топ обменников по доходу.
- Обменники: `DataTable` на **живых данных** (поиск, фильтр по статусу, выбор
  строк, смена статуса через меню) и редактор обменника — вкладки
  Профиль / Условия / Парсер / Логи, тестовый прогон парсера, просмотр логов.
- Валюты (справочник + очередь маппинга неизвестных валют), направления,
  парсеры (здоровье и конфигурация).
- Модерация отзывов, арбитраж жалоб, выплаты, партнёры (живой список),
  экспорт данных + аудит-лог.
- Разделы без бэкенда (доход, клики, здоровье парсеров, модерация, жалобы,
  выплаты, аудит) используют **явно помеченные демо-данные**; каталог
  (обменники / валюты / направления) — из живого API.

Backend:

- CORS расширен для внутренних кабинетов: разрешены заголовок `Authorization` и
  методы `POST` / `PATCH` / `DELETE` (нужно админке и партнёрскому кабинету).

### Аккаунты, алерты и уведомления (🌐 `kurso-web`)

- Личный кабинет (лейаут с сайдбаром): обзор, алерты, избранное, история
  переходов, отзывы, настройки, партнёрка — на реальном клиентском состоянии с
  персистентностью и пустыми состояниями (без демо-заглушек).
- Ценовые алерты: конструктор создаёт алерт на **живых курсах** (порог считается
  от текущего лучшего курса); алерты сохраняются, ставятся на паузу и удаляются.
  Экран срабатывания показывает лучший обменник и альтернативы выше порога.
- Избранное и история переходов — настоящие: сердечко на карточках обменников и
  каждая кнопка «Перейти» наполняют кабинет.
- Вход / регистрация подключены к реальной авторизации пользователей (см. раздел
  ниже); middleware `auth` защищает кабинет, в шапке — аватар пользователя.
  Онбординг — превью-прогон, сессию не создаёт.

### Реальная авторизация админки (🔧 `kurso-api`, ⚙️ `kurso-admin`)

- Бэкенд: проверка пароля (bcrypt), пара JWT — короткий access и длинный refresh
  (HS256); эндпоинты `/admin/auth/login|refresh|logout|me`, middleware
  `RequireAdmin`, сид-администратор при старте.
- Access-токен хранится только в памяти клиента; refresh — в **httpOnly-cookie**,
  которую читает сервер (токенов в localStorage нет). Прозрачный
  401 → refresh → повтор запроса; сессия восстанавливается при загрузке.
- Каталожные чтения админки переведены на **Pinia Colada** (общие
  дедуплицированные запросы); CORS с учётом cookie-credentials.

### Отзывы и модерация (🔧 `kurso-api`, 🌐 `kurso-web`, ⚙️ `kurso-admin`)

- Отзывы: публичные `GET` (опубликованные + сводка рейтинга и гистограмма) и
  `POST` с базовым автомодератором — чистые отзывы публикуются сразу, спорные
  уходят в очередь; публикация пересчитывает рейтинг обменника.
- Жалобы на отзывы: `POST` жалобы + админские `GET` / `PATCH` для разбора.
- kurso-web: блок отзывов на странице обменника рендерится через `useAsyncData`
  (SSR/ISR) — сводка, гистограмма, фильтры, форма отправки, кнопка «Пожаловаться».
  Чтения на клиенте — нативными средствами Nuxt; Pinia Colada только для действий.
- kurso-admin: очередь модерации читает реальные ожидающие отзывы и публикует
  или отклоняет их через API.

### Реальная авторизация пользователей и данные Nuxt (🔧 `kurso-api`, 🌐 `kurso-web`)

- Бэкенд: реальные пользователи (bcrypt) — `POST /api/v1/auth/register|login`,
  `refresh`, `logout`, защищённый `me` и не-ротирующий `session` для SSR;
  middleware `RequireUser`. Auth-сервис обобщён (`Account`/`Repo`) — один тип
  обслуживает и админов, и пользователей с разными секретами.
- Токены как у админки: access — только в памяти клиента, refresh — в
  **httpOnly-cookie** (путь `/`, настраиваемый домен), которую читает и валидирует
  сервер.
- **Учёт особенностей Nuxt**: плагин резолвит сессию на сервере (пробрасывает
  cookie в `/session`), поэтому приватные страницы рендерятся на SSR с правильным
  пользователем без «мигания» гидратации; access-токен подтягивается на клиенте.
  Соцвходы (Telegram/Google/Apple) честно помечены как «скоро», а не сфабрикованы.
- Чтения каталога и курсов в kurso-web переведены с Pinia Colada на `useAsyncData`
  (SSR/ISR); живые курсы патчат общий `useState`-стор из SSE. Модуль
  `@pinia/colada-nuxt` убран — в клиенте Colada остаётся только под действия.

## [0.1.0] — 2026-07-04 — Фундамент

Инфраструктурная основа: рабочее окружение, схема БД и контракт API. Кода фич нет.

### Общее / монорепо

- Монорепо из четырёх проектов: `kurso-api` (Go), `kurso-web` (Nuxt), `kurso-admin`
  и `kurso-partner` (Vue).
- Корневой `README.md` и README в каждом проекте.
- `.gitignore` (в т.ч. `docs/PLAN.md` не версионируется).

### Backend (`kurso-api`)

- Go-модуль, точка входа `cmd/kurso/main.go`, конфигурация через env, `slog`-логгер
  (text в dev / json в prod), graceful shutdown.
- Гексагональная структура `internal/`: `domain` (ядро и порты), `service`
  (сценарии), `adapter/http` (входящий HTTP-адаптер на chi), `platform`
  (logger, Postgres через pgx/pgxpool, Redis через go-redis, HTTP-сервер).
- Эндпоинты: `GET /healthz`, `GET /readyz` (проверка Postgres + Redis),
  `GET /api/v1/currencies` (заглушка).
- Multi-stage `Dockerfile` (distroless runtime).
- Unit-тесты: `config`, health-handler.

### База данных

- 11 миграций goose со всеми базовыми таблицами (20 шт.): `users`, `admins`,
  `exchanger_users`, `exchangers`, `currencies`, `currency_aliases`, `directions`,
  `rates`, `rates_history` (партиционирована по `recorded_at`), `parser_configs`,
  `alerts`, `triggered_alerts`, `push_subscriptions`, `reviews`, `review_replies`,
  `review_reports`, `clickouts`, `partner_payouts`, `admin_audit_log`,
  `security_events`.
- goose подключён как Go-tool (без отдельной установки).

### Контракт и кодогенерация

- `api/openapi.yaml` — единый источник правды (OpenAPI 3.0).
- oapi-codegen генерирует Go-типы; openapi-typescript — TS-типы во всех трёх
  фронтендах. Регенерация — `make generate`.

### Фронтенды

- `kurso-web`: Nuxt 3 + TypeScript + Tailwind.
- `kurso-admin`, `kurso-partner`: Vite + Vue 3 + TypeScript.
- ESLint + Prettier в каждом; сгенерированные типы API из общего контракта.

### Инфраструктура и DX

- Локальная разработка идёт на **хостовых Postgres/Redis** (`localhost:5432` /
  `6379`), без Docker. `make db-create` заводит роль и БД `kurso` в локальном
  Postgres.
- `docker-compose.yml` — **прод-стек** (self-host / прод-паритет) целиком:
  Postgres 16 + Redis 7 + `api` + три фронтенда (прод-образы: `kurso-web` —
  Nuxt SSR на node, `kurso-admin` / `kurso-partner` — статика в nginx со
  SPA-fallback). Управление — `make stack-up` / `stack-down`; порты хоста
  переопределяемы через `POSTGRES_PORT` / `REDIS_PORT`.
- `Makefile`: `dev` (API + 3 фронта разом), `run`, `migrate`, `generate`, `test`,
  `lint`, `build`, `db-create`, `stack-up`/`stack-down` и др.
- Pre-commit: husky + lint-staged в корне монорепо (один `core.hooksPath`);
  `gofmt`/`go vet` для Go, `eslint --fix`/`prettier --write` для фронтендов.

### CI/CD

- GitHub Actions `ci.yml`: бэкенд (gofmt, vet, тесты, сборка, проверка
  актуальности кодогена) + фронтенды матрицей (lint, build).
- `docker.yml`: сборка Docker-образа бэкенда (без пуша). Деплой — вручную.
